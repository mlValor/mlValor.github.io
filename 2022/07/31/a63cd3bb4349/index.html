Recorder插件

发表于2022-07-31|更新于2022-08-15|Unity插件
|字数总计:1.9k|阅读时长:6分钟|阅读量:

一.简介

注意： Recorder只能在 Unity Editor 中使用。

Unity Recorder 允许您在Play模式期间从Unity 编辑器捕获和保存数据：动画、视频、图像、音频等。

相对于直接录屏的优点：

  • 可以直接录Game视图，还可以录不同相机的视图（需要给相机设置Tag）

  • 可以自定义输出分辨率，不再受限于屏幕的分辨率

  • 支持多种类型的输出（Animation Clip、视频、序列帧，GIF等）

缺点：

  • 不能像一些录屏软件那样，边录屏边对屏幕进行图画（进行标记，写字啥的），只能后期处理

  • 默认情况下录不了鼠标（需要手动调用API：Cursor.SetCursor(<texture>, <hotspot>, <CursorMode>)，才能看见鼠标）

  • 没做特殊处理的情况下不能录制外部音源

二.快速上手

1.简单使用

①.首先要打开录制面板：菜单栏 -> Window -> General -> Recorder -> Recorder window

②.点击Add Recorder，添加一个录制类型，这里选择Movie。

③.点击**START RECORDING按钮，或着Play按钮来进行视频录制。一般没特殊需求，使用默认配置**即可。

④.视频的输出目录，可以通过点击**蓝色边框区域快捷按钮**开启。

2.设置录制源

Recorder录屏有多种不同的录制方式：

Game View：录制源为整个Game视窗

Targeted Camera：将录制源改成场景相机

  • ActiveCamera：使用当前Acitve的相机
  • MainCamera：使用tag为MainCamera的相机
  • TaggedCamera：使用与GameObject标签匹配的第一个相机

360 View（部分太抽象，用的机翻）：

  • Output Dimensions：输出尺寸，360 度视频的宽度和高度（以像素为单位）
  • Cube Map Size：立体贴图大小，360 度视频的立方体贴图的边长（以像素为单位）
  • Record in Stereo：立体音响，分别录制 360 度视频的左右立体视图。Recorder将两个视图包含在同一图像中：左视图位于顶部，右视图位于底部
  • Stereo Separation：立体声分离，所选摄像机 Y 轴上左右视图之间的角度

Render Texture Asset：将录制源设置为指定的RenderTexture

  • Render Texture：将需要录制的RenderTexture拖入即可

Texture Sampling：

  • Aspect Ratio：宽高比，输出的宽高比
  • Supersampling Grid：超级采样网格，用于超级采样模式的子像素网格的大小。
  • Rendering Resolution：渲染分辨率，采样输入的分辨率
  • Output Resolution：输出分辨率，采样输出的分辨率

其他参数说明:

  • Include UI：在录制中包含UI对象（只有ActiveCamera模式才有用）
  • Filp Vertical：垂直翻转最终输出的视频

3.设置录制的分辨率

Recorder可以通过调整Cpature下拉面板里的Output Resolution参数来设置不同的分辨率。

默认状况下是：Match Window Size（匹配当前Game视图的分辨率）,如果有特殊的分辨率需求，除了官方提供好的几个选项外，可以选择Custom来自定义分辨率参数。

4.设置录制的视频格式以及质量

Media File Format：媒体文件格式，记录输出的文件编码格式，默认为H265.MP4

Quality：输出视频的质量，有中，高，低，三种选择

5.设置导出的文件名称

File Name：输出文件的名称。主要包括普通文本和动态占位符（使用“+Wildcards（通配符）”按钮）。

Path：Recorder保存输出文件的文件夹路径，目录的绝对路径会直接显示在下一行。这里简单介绍下路径选项。

红色区域：根目录选项，指定输出路径相对于哪个根位置的选项（或者该路径是绝对路径）

  • Project：项目文件的相对路径（Assets目录的父级）
  • Assets Folder：Assets目录下的相对路径
  • Streaming Assets：StreamingAssets目录下的相对路径
  • Persistent Data：PersistentData目录下的相对路径
  • Temporary Cache：Temporary Cache目录下的相对路径
  • Absolute：绝对路径

绿色区域：子文件夹名称，如果为空，则会直接创建在上面**红色区域所指定的目录下。不为空，则会输出到红色区域**所指定的目录下对应名称的文件夹。

Take Number： Recorder用于对记录进行编号的值。 每次录制后增加1。与**<Take>通配符关联**。

6.其他说明

①.录制类型介绍

Unity Recorder 默认包括以下Recorder类型：

  • Animation Clip：生成 Unity 动画格式（.anim 扩展名）的动画剪辑。

  • Movie：生成 H.265 MP4、H.264 MP4、VP9 WebM 或 ProRes QuickTime 格式的视频。

  • Image Sequence：生成 JPEG、PNG 或 EXR (OpenEXR) 格式的图像文件序列。

  • GIF Animation：生成动画 GIF 文件。

  • Audio：生成 WAV 格式的音频剪辑。

②.录制面板介绍

Play按钮：开始录制视频。如果尚未运行场景，它会自动播放当前场景

START RECORDING：与Play按钮功能相同

Exit Play Mode：录制停止时退出播放模式

Recording Mode：指定录制固定帧或录制固定的时间段

  • Manual：手动录制模式，记录开始和停止之间的每一帧
  • Single Frame：根据指定的帧号记录一帧（比如说，填10，那么就会在开始录制之后的第10帧停止，并且只会记录第10帧，不会记录前几帧，有点类似截屏）
  • Frame Interval：根据指定的起始帧号和结束帧号记录帧间隔内的所有帧（起始帧->终止帧，如果填【10，20】，那么会在开始录制的第10帧开始记录，到20帧后录制停止）
  • **Time Interval(sec)**：根据指定的开始时间和结束时间记录一个时间间隔内的所有帧（起始秒->终止秒）

![Recorder-Recording Mode](Recorder/Recorder-Recording Mode.png)

Playback：录制时使用的帧率类型：Constant（恒定的）或Variable（变化的）

  • Constant：即使实际帧速率较低或较高，帧速率在录制过程中也不会发生变化
  • Variable：应用程序的帧速率，在录制过程中可能会有所不同。 并非所有Recorder都支持此选项

TargetFPS：用于录制的恒定帧速率，与捕获源的帧速率无关（可以理解为每秒录制的帧数）

Cap FPS：当捕获源的帧速率快于目标帧速率时，限制它的帧速率

③.Animation Clip参数说明

Capture面板：

GameObject：需要录制Animation Clip的对象

Recorded Components：要录制的游戏组件（GameObject上的），可以选择多个组件

Record Hierarchy：将目标对象的所有子物体一起进行录制

Clamped Tangents：启用后，将生成的动画关键点切线设置为 ClampedAuto，否则设置为 Auto（旧版）

Anim.Compression：用来压缩录制的动画曲线数据的关键帧缩减水平

④.Image Sequence参数说明

Format面板：

Media File Format：记录输出的文件编码格式

Include Alpha：图片中包括Alpha通道。在高清渲染管道（HDRP）中，你需要将缓冲区格式设置为R16G16B16A16

Compression：使用EXR格式时要应用的数据压缩方法

Color Space：在输出图像中使用的色彩空间（gamma曲线，gamut）。如果你选择了一个选项来获得非箝制值，你必须： （1）使用高清渲染管道（HDRP）； （2）在你的场景中禁用任何色调映射； （3）在选定的相机上禁用抖动。

⑤.GIF Animation参数说明

Format面板：

Quality：GIF图片的采样精度，值越高图片越清晰

Frame Rate：GIF帧率

Loop：是否循环

⑥.如何录制鼠标

录制鼠标需要手动调用Unity API接口。

⑦.在开着Recorder窗口的时候，可以用快捷键F10开启录制
