Timeline功能扩展(EventMarker)——自定义Timeline的帧事件功能

发表于2021-08-22|更新于2022-08-11|UnityPlayable
|字数总计:2.3k|阅读时长:12分钟|阅读量:

功能需求

Runtime模式：

  1. Marker ID：Marker标识

  2. Jump To：需要能够切换到当前轨道的其他时间点

    • None：不做任何处理
    • Jump To Marker：跳到当前轨道除自己外的其他标记上
      • Jump Marker：指定的跳转标记，只在Jump To Marker模式生效（在Editor类处理赋值）
    • Jump To Time：跳到当前轨道上指定的时间
      • Jump Time：指定的跳转时间，只在Jump To Time模式生效
    • Jump Count：指定最大跳转次数，小于0则忽略

  3. Set Speed：设置当前轨道的播放速度

    • Speed：当Speed为0时，则轨道暂停，为1则继续播放

  4. Execute Event：执行事件回调

    • None：不做任何处理
    • Unity Event：只执行 Unity Event的事件监听系统
      • Target Event Index：标记当前Marker对应的Unity Event下标（在Editor类处理赋值）
    • Action：只执行Runtime模式下注册的事件回调
      • String Key：用于调用运行时注册的事件key（不能为null或者empty）
    • Everything：Unity Event跟Action同时执行
    • Execute Event Count：执行事件回调的最大次数，小于0则忽略

Editor模式：

  1. EventMarkerReceiver脚本图标替换：主要利用反射调用，将图标样式设置成SignalReceiver Icon
  2. EventMarker脚本图标替换：同样利用反射调用，将图标样式设置成SignalEmitter Icon
  3. Timeline轨道上的Marker样式替换：利用特性[CustomStyle("SignalEmitter")]使用SignalEmitter的USS样式
  4. 重写EventMarker的InspectorGUI，优化显示
