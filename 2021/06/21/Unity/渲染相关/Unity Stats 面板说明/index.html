Unity Stats 面板说明

发表于2021-06-21|更新于2022-08-11|Unity渲染相关
|字数总计:348|阅读时长:1分钟|阅读量:

![1-1](Unity Stats 面板说明/1-1.png)

1.Time per frame and FPS(每帧的时间和FPS):

处理和渲染一个游戏帧（以及由此产生的FPS）所花费的时间。请注意，此数字仅包括进行帧更新和渲染 Game 视图所用的时间；不包括在 Editor 中绘制 Scene 视图、检视面板处理和其他仅限于 Editor 的处理所花费的时间。

2.Batches(绘制调用):

“批处理 (Batching)”可让引擎尝试将多个对象的渲染组合到一个内存块中以便减少由于资源切换而导致的 CPU 开销。

3.Saved by batching(合并的批次数):

为确保良好的批处理，应尽可能在不同对象之间共享材质。更改渲染状态会将批次分成具有相同状态的组。

4.Tris and Verts(三角面和顶点数):

绘制的三角形和顶点的数量。在针对低端硬件进行优化时，这一点非常重要。

5.Screen(屏幕):

屏幕大小以及抗锯齿级别和内存使用情况。

6.SetPass(Pass通道设置):

渲染 pass 的数量。每个 pass 都需要 Unity 运行时绑定一个新的着色器，这可能会带来 CPU 开销。

7.Visible Skinned Meshes(可视蒙皮网格):

渲染的蒙皮网格的数量。

8.Animations(动画):

播放的动画的数量。
